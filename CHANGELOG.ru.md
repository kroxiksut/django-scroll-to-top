# История изменений

Все заметные изменения проекта документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
проект следует [семантическому версионированию](https://semver.org/lang/ru/).
Пока версия остаётся в серии `0.x`, публичный API может меняться между
минорными релизами.

## [0.3.0] - 2026-07-01

### Добавлено

- Кнопка **«Создать стартовую конфигурацию»** в списке профилей: одним кликом
  создаёт и публикует профиль с ревизией по умолчанию для областей «сайт» и
  «админка». Свежая установка больше не показывает три пустые секции админки
  рядом с уже видимой на странице кнопкой. Действие идемпотентно и заполняет
  только те области, где ещё ничего не опубликовано.
- **Сворачиваемое меню в админке**: группу scroll-to-top в левой боковой панели
  админки можно свернуть небольшим переключателем (только её три пункта — другие
  приложения и футер-кнопка не затрагиваются), состояние запоминается в браузере.
  Это чистое прогрессивное улучшение — читает стандартный DOM админки и не
  переопределяет стили глобальных элементов.
- Человекочитаемые сообщения при нарушении правил уникальности вместо сырого
  «Нарушено ограничение „…“»: для дублирующихся профилей (один глобальный
  профиль на область, один профиль на область и сайт) и для второй
  опубликованной ревизии в одном профиле.
- Усиление цепочки поставок в CI: конфигурация Dependabot (pip, npm и GitHub
  Actions, еженедельно) плюс пиннинг всех GitHub Actions до commit-SHA с
  комментарием-версией — обновления экшенов теперь проходят ревью, а не «плывут»
  за подвижным тегом.

### Изменено

- «Живая» ревизия теперь определяется по **статусу** ревизии (единственный
  источник правды), а не по хранимому указателю `ScrollTopProfile.published_revision`.
  Взаимная привязка профиль↔ревизия убрана. Чтобы ревизия стала активной,
  достаточно опубликовать её действием **«Опубликовать выбранную ревизию»** —
  править профиль и вручную привязывать ревизию больше не нужно. Миграция данных
  сохраняет текущую активную ревизию на существующих установках, а частичный
  уникальный индекс гарантирует не более одной опубликованной ревизии на профиль.
- Форма добавления профиля больше не показывает бесполезное, всегда пустое поле
  «Опубликованная ревизия»; активная ревизия отображается ссылкой только для
  чтения, вычисленной по статусу.

## [0.2.0] - 2026-06-28

### Добавлено
- Описаны лимиты для загружаемых SVG-иконок: до 100 КБ на файл, 128 элементов,
  8 уровней вложенности и 20 000 символов данных пути на фигуру. Файлы больше
  лимита отклоняются, а не обрезаются.
- Дополнительные предупреждения о конфигурации: когда профиль ссылается на
  несуществующий сайт или включён, но ещё ничего не опубликовано.
- Сканирование безопасности в CI (анализ кода и проверка зависимостей в PR).
- Небольшие улучшения карточки на PyPI (ссылка на release notes, отметка
  OS-independent).

### Изменено
- Расширена тестовая матрица Django: 4.2 LTS, 5.0, 5.1, 5.2 LTS и 6.0 — каждая
  версия и устанавливается, и покрыта CI.
- Более мягкая и точная формулировка про доступность (спроектировано под WCAG
  2.2 AA; полный независимый аудит ещё запланирован).
- Более дружелюбный быстрый старт в README: установить, выполнить миграции,
  добавить один тег — в админку заходить не обязательно.

### Исправлено
- Не все подписи в админке были помечены как переводимые, поэтому часть всегда
  отображалась по-английски независимо от выбранного языка (детали загруженной
  иконки и метки «создано/обновлено»). Теперь они переводимы и переведены на
  русский, а каталог переводов обновлён.

## [0.1.0] - 2026-06-27

Первый публичный релиз (бета) **django-scroll-to-top** — переиспользуемого
Django-приложения, которое отображает один доступный настраиваемый элемент
«наверх» на страницах сайта и в стандартной Django Admin.

Полный набор возможностей и настройку см. в [README](README.ru.md) и
[документации](docs/README.ru.md). Начиная с этого релиза файл фиксирует
изменения между версиями, а не пересказывает, что делает пакет.
