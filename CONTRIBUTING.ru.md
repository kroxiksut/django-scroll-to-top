# Участие в разработке

Спасибо за интерес к улучшению `django-scroll-to-top`.

## Сообщения об ошибках и предложения изменений

- **Ошибки и идеи:** откройте GitHub issue с понятным описанием; для ошибок
  приложите минимальное воспроизведение и версии Django и Python.
- **Уязвимости безопасности:** не открывайте публичный issue — следуйте
  [SECURITY.ru.md](./SECURITY.ru.md).
- **Pull request'ы:** держите их сфокусированными, описывайте изменение,
  добавляйте тесты при изменении поведения, обновляйте документацию и добавляйте
  запись в раздел `## [Unreleased]` в [CHANGELOG.ru.md](./CHANGELOG.ru.md).
  Убедитесь, что линт, проверка типов и тесты проходят.
- Участвуя, вы соглашаетесь с [Кодексом поведения](./CODE_OF_CONDUCT.ru.md).

### Линтинг JavaScript

Браузерные ассеты проверяются ESLint. Node — инструмент только для разработки:
он не является зависимостью рантайма и не входит в собираемый пакет.

```bash
npm ci           # однократно, ставит ESLint + TypeScript в node_modules/ (в .gitignore)
npm run lint     # проверяет статический JS (сгенерированные *.min.js игнорируются)
npm run typecheck  # проверяет публичный контракт *.d.ts через tsc --noEmit
npm run check    # оба шага сразу
```

`scroll-to-top.d.ts` / `obstacle-adapter.d.ts` описывают публичный JS-контракт
(`window.djstt`, `window.djsttObstacleAdapter` и события `djstt:*`). Держите их
в синхроне при изменении этой поверхности; `npm run typecheck` проверяет только
декларации, а не ES5-реализацию.

При изменении `scroll-to-top.js` или `obstacle-adapter.js` пересобирайте
минифицированные ассеты через `python tools/minify_assets.py` в том же change set.

Перед изменением публичного поведения прочитайте `AGENTS.md`, `AI_RULES.md` и
`ARCHITECTURE.md`. Держите изменения сфокусированными и добавляйте тесты при
изменении поведения.

Код, исходные строки, примеры и каноническая документация пишутся на
английском. Пользовательские строки сразу заворачиваются в Django i18n. Затем
в том же завершённом change set обновляется русский `.po` и пересобирается
`.mo`. Сначала обновляется английская документация, затем синхронизируется
русская.
