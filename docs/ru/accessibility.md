# Доступность

- [Назад к индексу документации](../README.ru.md)
- [Поведение и браузерный слой](./runtime.md)

Элемент ориентирован на WCAG 2.2 AA в рамках своей области. Структурный контракт
ниже реализован и покрыт тестами (`tests/test_accessibility.py`).

## Реализовано и покрыто тестами

- Переводимое **доступное имя** на самом элементе и на элементе закрытия,
  независимое от иконки или подсказки (`aria_label` переопределяет переведённое
  значение по умолчанию).
- Минимальный **порог цели 24×24 CSS px** независимо от заданного размера.
- **Запасной вариант-ссылка без JavaScript** — обычная ссылка к началу документа.
- **Отсутствие положительного `tabindex`**, отсутствие ловушки клавиатуры и
  неожиданных перемещений фокуса.
- Видимые контуры **`:focus-visible`** (настраиваемые цвет, ширина и отступ
  кольца фокуса).
- **`prefers-reduced-motion`**: пользователям с reduced-motion всегда даётся
  мгновенный переход, без плавной анимации, независимо от `scroll_behavior`.
- Режим **forced-colors / высокого контраста** нейтрализует любой визуальный
  вариант.
- **RTL-безопасные** логические свойства для размещения и отступов.
- Декоративный SVG скрыт от API доступности; смысл несёт доступное имя.

## Контраст: рекомендательный, не навязываемый

Контраст цветов **не** навязывается при сохранении — оператор может выбрать любые
цвета. Низкоконтрастные сочетания показываются как непрерывающее предупреждение
админки и могут быть проверены командой:

```console
python manage.py scroll_to_top_check_contrast
```

Команда завершается с ненулевым кодом, если опубликованная ревизия не проходит
проверку контраста, поэтому ею можно гейтить развёртывание, не блокируя
конфигурацию.

## Запланировано на потом

Полный аудит WCAG 2.2 AA и проверка масштабирования (200% / 400%) в реальных
браузерах отслеживаются как более поздний этап стабилизации, а не как часть
текущего автоматического контракта.
