# Демо-проект

- [Назад к индексу документации](../README.ru.md)
- [Поведение и браузерный слой](./runtime.md)

Автономный проект Django для локального демо и ручного тестирования находится в
`demo/` (небольшое приложение `library` с наполненным контентом и настроенным
базовым шаблоном). Именно на нём проверяется сквозная интеграция.

![Элемент по умолчанию на демо-сайте рядом с боковой панелью](../assets/shared/default-button-demo-site.png)

## Запуск

```console
python demo/manage.py migrate
python demo/manage.py runserver
```

Первый `migrate` наполняет:

- рабочего суперпользователя `admin` / `admin`;
- опубликованную конфигурацию «прокрутка наверх» по умолчанию для публичного
  сайта;
- опубликованную конфигурацию «прокрутка наверх» по умолчанию для админки Django.

## Что демонстрируется

- Добавление приложения к уже работающему сайту и одиночная вставка тега в
  базовый шаблон.
- Подключение URLConf пакета для эндпоинта таблицы стилей при строгом CSP.
- Регрессия обычных страниц и админки после изменения конфигурации.

### Страница препятствий

Страница `Obstacles` (`/obstacles/`) демонстрирует движок столкновений и
необязательный адаптер препятствий: cookie-баннер в правом нижнем углу с
кнопками открытия/сворачивания/закрытия, виджет чата, тост и липкую мобильную
навигацию как одновременные препятствия. Кнопкой **Toggle collision debug**
можно визуализировать прямоугольники препятствий (эквивалент
`window.djstt.debug(true)`).

## Локализация в демо

- Публичные страницы только на английском.
- Админка Django использует встроенную английскую и русскую локализацию Django.
- Демо-контент оригинален и предназначен для локального тестирования прокрутки и
  вёрстки.
