## Системные проверки

Пакет регистрирует системные проверки Django (`checks.py`) со стабильными
идентификаторами и практичными подсказками. Они выполняются командой
`python manage.py check` и при обычном старте:

| Id | Сообщает о |
| --- | --- |
| `dstt.W001` | Невозможно проинспектировать миграции. |
| `dstt.W002` | Миграции не применены. |
| `dstt.W003` | Неподдерживаемый `CSP_MODE`. |
| `dstt.W004` | URLConf пакета отсутствует, а отрисовка на сайте включена. |
| `dstt.W005` | `django_scroll_to_top` стоит после `django.contrib.admin`. |
| `dstt.W006` | Неподдерживаемый `DEFAULT_COLLISION_POLICY`. |
| `dstt.W007` | `DJANGO_SCROLL_TO_TOP` не словарь или содержит неизвестные ключи. |
| `dstt.W008` | Задан `ADMIN_ENABLED`, но `django.contrib.admin` не установлен. |
| `dstt.W009` | Задан `SITES_FRAMEWORK_ENABLED`, но `django.contrib.sites` не установлен. |
| `dstt.W010` | Отсутствуют шаблоны пакета или минифицированные статические ассеты. |

## Management-команды

```console
python manage.py scroll_to_top_diagnose
```

Печатает разрешённую конфигурацию по областям (без секретов) — удобно, чтобы
подтвердить, к какому профилю и ревизии приходит развёртывание.

```console
python manage.py scroll_to_top_check_contrast
```

Завершается с ненулевым кодом, если опубликованная ревизия не проходит
рекомендательную проверку контраста, поэтому ею можно гейтить развёртывание в CI,
не блокируя конфигурацию в админке.
