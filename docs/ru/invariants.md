Эти инварианты должны соблюдаться при разработке пакета. Это канонические
ограничения из `AI_RULES.md`, `ARCHITECTURE.md` и справочника основного скилла;
нарушение любого — регресс, даже если тесты ещё проходят.

## Публичный контракт

- Один тег шаблона — это весь публичный API сайта: `{% scroll_to_top %}` (с
  необязательным `scope="admin"`). Визуальных аргументов у тега нет.
- Стабильная публичная поверхность — `window.djstt`, DOM-события `djstt:*`,
  атрибуты `data-dstt-*`, маркер `data-scroll-top-obstacle` и шаблон
  `scroll_to_top.html`. Внутренние API сервисов/моделей приватны.

## Владение конфигурацией

- Настройки включают возможности (установка/инфраструктура); профили в базе
  данных владеют обычным представлением и поведением. Никогда не дублируйте
  внешний вид в настройках.
- Области «site» и «admin» независимы и никогда не разрешаются через одну
  неявную запись.
- Порядок разрешения: профиль сайта → глобальный → безопасные встроенные
  значения.
- Значения для десктопа первичны; каждое мобильное поле явно наследует или
  переопределяет.

## Отрисовка и жизненный цикл

- Рендерер возвращает типизированные, удобные для сериализации данные, никогда не
  ORM-объект.
- Конфигурация разрешается не более одного раза за запрос; отрисовка никогда не
  создаёт строки.
- Черновые и опубликованные ревизии редактируются на месте (редактирование
  опубликованной обновляет живую конфигурацию и инвалидирует кэш её области);
  архивные ревизии — неизменяемые снимки.
- Публикация, откат и инвалидация кэша — операции сервиса.

## Безопасность и изоляция

- Production никогда не отрисовывает несанитизированный загруженный SVG.
  Разбирайте как XML, никогда не регулярками. Отклоняйте
  DTD/сущности/скрипты/события/внешние ссылки/`foreignObject`.
- Снабжайте пространством имён каждый CSS-класс, ID, data-атрибут и DOM-событие.
  Никогда не стилизуйте глобальные элементы (`a`, `button`, `svg`, `body`). Без
  CDN, без шага сборки во время выполнения.
- По умолчанию без телеметрии; никогда серверного запроса к произвольному URL.
- Строгий CSP не должен молча требовать `unsafe-inline`.

## Браузерный слой и миграции

- Держите видимость / столкновения / прокрутку / скрытие как отдельные машины
  состояний.
- Миграции остаются детерминированными — без сети, без анализа живых страниц или
  вычисленных стилей, без файлов конкретного развёртывания.
