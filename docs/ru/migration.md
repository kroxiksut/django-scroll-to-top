- [Назад к индексу документации](./README.md)

Пакет поставляет собственные миграции Django (`migrations/0001_initial` и далее).
Применяйте их стандартной командой:

```console
python manage.py migrate django_scroll_to_top
```

## Детерминированные миграции

Миграции остаются детерминированными: они никогда не инспектируют живые
страницы, вычисленные стили, сетевые ресурсы или файлы конкретного развёртывания.
Их запуск даёт одинаковую схему независимо от окружения, поэтому их безопасно
выполнять в CI и автоматических развёртываниях.

## Обновление

- Новые поля представления/поведения вводятся через обычные миграции схемы;
  существующие ревизии сохраняют свои значения, а новые поля принимают значения
  по умолчанию из модели.
- Опубликованные и архивные ревизии сохраняются при обновлениях. Архивные ревизии
  — неизменяемые снимки, поэтому откат после обновления восстанавливает точную
  историческую конфигурацию.
- После обновления выполните `python manage.py migrate`, затем
  `python manage.py check`, чтобы системные проверки `dstt.W001` / `dstt.W002`
  подтвердили, что миграции инспектируемы и применены.

## Краткий обзор модели данных

- `ScrollTopProfile` — область, необязательный `site_id`, `is_enabled`, указатель
  на опубликованную ревизию.
- `ScrollTopRevision` — полный снимок внешнего вида/поведения со статусом
  `draft` / `published` / `archived`.
- `ScrollTopUploadedIcon` — санитизированный SVG плюс метаданные
  лицензии/атрибуции.

Жизненный цикл см. в [operations-admin.md](./operations-admin.md), а полную карту
модуля — в [overview.md](./overview.md).
