# Быстрый старт

- [Назад к индексу документации](../README.ru.md)
- [Обзор и текущее состояние](./overview.md)

Это руководство проводит `django-scroll-to-top` от чистой установки до
отрисованного элемента «прокрутка наверх» на сайте и в стандартной админке
Django. Полный справочник настроек и конфигурацию в админке см. на страницах
конфигурации и админки (запланированы) и в [обзоре проекта](../../README.ru.md).

> **Замечание о статусе до релиза.** Дистрибутив ещё не опубликован на PyPI,
> поэтому команда `pip install django-scroll-to-top` ниже описывает планируемый
> контракт. До первого релиза устанавливайте из локальной копии (например,
> `pip install -e .` из корня репозитория).

## 1. Установка

```console
python -m pip install django-scroll-to-top
```

Целевая совместимость: Python 3.10+ и Django 4.2 LTS, 5.x и 6.x
(`Django>=4.2,<7`).

## 2. Подключение приложения

Добавьте приложение в `INSTALLED_APPS`. Если включена интеграция с админкой,
поместите `django_scroll_to_top` **перед** `django.contrib.admin`, чтобы
переопределение подвала `admin/base_site.html` из пакета выбиралось обычным
разрешением шаблонов:

```python
INSTALLED_APPS = [
    "django_scroll_to_top",   # перед django.contrib.admin
    # ...
    "django.contrib.admin",
]
```

## 3. Настройка (настройки отвечают только за установку/инфраструктуру)

Конфигурация задаётся через словарь `DJANGO_SCROLL_TO_TOP`. Все ключи
необязательны; приведённые значения — встроенные значения по умолчанию:

```python
DJANGO_SCROLL_TO_TOP = {
    "SITE_ENABLED": True,              # отрисовка через {% scroll_to_top %}
    "ADMIN_ENABLED": True,             # внедрение в стандартную админку Django
    "SITES_FRAMEWORK_ENABLED": False,  # включить профили по Site
    "CSP_MODE": "external",            # "external" | "nonce"
    "ADMIN_SHOW_ON_AUTH_PAGES": False, # показывать на странице входа / сброса пароля админки
    "DEFAULT_COLLISION_POLICY": "ignore",  # ignore | shift | fallback_corner | hide
}
```

Внешний вид и поведение здесь **не** настраиваются — они хранятся в конфигурации
на основе базы данных и редактируются через админку Django (см. шаг 7).

## 4. Подключение URL

Подключите URLConf пакета, чтобы элемент мог загрузить свой версионированный
эндпоинт таблицы стилей того же источника (необходимо для пути со строгим CSP):

```python
# urls.py
from django.urls import include, path

urlpatterns = [
    # ...
    path(
        "scroll-to-top/",
        include(
            ("django_scroll_to_top.urls", "django_scroll_to_top"),
            namespace="django_scroll_to_top",
        ),
    ),
]
```

## 5. Миграции

```console
python manage.py migrate
```

## 6. Отрисовка элемента

Загрузите библиотеку тегов и отрисуйте элемент один раз ближе к концу общего
базового шаблона:

```django
{% load scroll_to_top %}

<!-- Содержимое страницы -->

{% scroll_to_top %}
```

Пока конфигурации в БД нет, тег отрисовывается из безопасных встроенных значений.
Базовый вариант без JavaScript — обычная ссылка к началу документа в выбранном
углу; обход столкновений, темизация и скрытие — это прогрессивные улучшения.

Для собственного `AdminSite` или переопределённого `admin/base_site.html`
добавьте элемент для админки явно в блок подвала:

```django
{% extends "admin/base.html" %}
{% load i18n scroll_to_top %}

{% block footer %}
  {{ block.super }}
  {% scroll_to_top scope="admin" %}
{% endblock %}
```

## 7. Настройка через админку Django

Откройте админку Django и отредактируйте конфигурацию «прокрутка наверх»:

- Области «site» и «admin» независимы; настраивайте каждую отдельно.
- Отредактируйте черновую ревизию, посмотрите живой предпросмотр и опубликуйте.
  Публикация архивирует предыдущую опубликованную ревизию; можно откатиться или
  клонировать ревизию в новый черновик.
- Значения для десктопа первичны; каждое поле, поддерживающее мобильную версию,
  явно наследует значение десктопа или хранит переопределение.

## 8. Проверка развёртывания

```console
python manage.py scroll_to_top_diagnose        # разрешённая конфигурация по областям, без секретов
python manage.py scroll_to_top_check_contrast  # ненулевой код выхода, если опубликованная ревизия не проходит
```

Системные проверки (`dstt.W001`–`dstt.W010`) сообщают о типичных ошибках
конфигурации: непримененные миграции, отсутствующий URLConf, неподдерживаемый
`CSP_MODE` или `django_scroll_to_top` после `django.contrib.admin`.
