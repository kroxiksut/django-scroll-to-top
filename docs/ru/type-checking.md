# Проверка типов (Pyright)

- [Назад к индексу документации](./README.md)
- [Тестирование пакета](./testing.md)

Пакет типизирован (поставляется `py.typed`, объявлен классификатор
`Typing :: Typed`). Pyright — эталонный проверщик типов; публичный payload
рендерера, аксессоры настроек и сигнатуры сервисов типизированы, поэтому
интеграторы получают стабильный, проверяемый контракт.

## Запуск Pyright

```console
python -m pyright
```

Локальные для пакета умолчания лежат в `pyrightconfig.json`. Установите dev-extras,
чтобы получить Pyright и Django-стабы:

```console
python -m pip install -e ".[dev]"
```

(`[dev]` даёт `ruff`, `pyright`, `django-stubs`, `build` и `twine`.)

## Почему это обязательно

- Рендерер возвращает типизированный, удобный для сериализации `RenderPayload` /
  `RenderContext`, а не ORM-объект, поэтому его форма должна оставаться
  проверяемой.
- Аксессоры настроек предоставляют типизированные значения (`CspMode`,
  `CollisionPolicy`, …), используемые системными проверками и рендерером.
- Браузерный рантайм поставляет типизированный контракт в `scroll-to-top.d.ts` и
  `obstacle-adapter.d.ts`, синхронизированный с рантаймом на чистом JS.
